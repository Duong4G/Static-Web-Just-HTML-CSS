Khái niệm Dependency Injection


Nguyên lý của Dependency Injection là:

  1. Các module không giao tiếp trực tiếp với nhau mà thông qua interface. Module cấp cao sẽ gọi module cấp thấp thông qua interface.
  2. Việc khởi tạo các module cấp thấp sẽ do DI Container thực hiện.
  3. Việc module nào gắn với interface nào sẽ được định nghĩa trong code hoặc trong file XML.
  4. Dependency Injection dùng để làm giảm sự phụ thuộc giữa các module, dể dàng hơn trong việc thay đổi các module, bảo trì code và testing.
Vd: Ta có một module cấp cao là Drawing(dùng để vẻ) và một module cấp thấp là Circle(hình tròn). Thông thường ta sẽ khởi tạo đối tượng Circle ngay trong Drawing mà vì vậy mà Drawing sẽ bị phụ thuộc vào Circle. Khi dùng DI thì Drawing sẽ gọi Circle thông qua interface là Shape.Module (Class )Circle sẽ gắn với Shape. Khi đó nếu ta thay đổi code trong Circle sẽ không ảnh hưởng trực tiếp đến module Drawing.
